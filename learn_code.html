    • Why do I want to code more?

    Here are MY top 3 reasons:

    1) Programming is versatile. Once you know 1 language, learning others is much less intimidating and many foundations overlap. A primary belief I hope to hold: once I self-learn a new program in detail (or expand my savviness in a program I know), I have already proven to myself that I can self-learn other programs of personal interest.

    2) I want to attain hard skills to expand my technical knowledge, to appeal to employers, and ultimately to comfortably understand the ICT back-end of health technologies.

    3) Opportunity for innovation and/or improving coding solutions. Perhaps in the future I will have an innovative idea for digital health and perhaps coding will offer me comfort to jump start it.


    My Approach to Self-Learn Code

    Before you start a course, I want to stress the importance of knowing why you want to code and what you want to prioritize. I started off with a list of programs I know (and how I can extend on these) versus what I want to know, but simply choosing one of the languages I want to learn for my first course did not work for me. I started off learning Java, got 30 minutes into a video, then realized I might want to change gears. Would it be in my best interest to learn Java from scratch or extend my comfort in Python into machine learning (ML)? This is a decision YOU make based off YOUR short-term vision for yourself. Which skills will you need first? For myself, although I intend to gain experience with Java, I realized I wanted to switch to a course surrounding ML in Python because I can see myself applying to practicums involving ML in healthcare in the coming months.

    Another important factor to consider before you self-learn code is how you want to learn. Although, you can pay for a course from many useful platforms like Udemy, CodeAcademy, and so forth; I wanted to start with a more affordable and "trial and error" route for my first course.
